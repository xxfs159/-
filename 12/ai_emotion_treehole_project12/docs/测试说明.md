# 测试说明

## 一、功能测试

### 测试1：压力/焦虑输入

输入：

```text
最近期末考试和项目DDL挤在一起，我总觉得来不及，晚上睡不着。
```

预期结果：

- 情绪识别为“压力”或“焦虑”
- 显示置信度和判断依据
- 给出拆分任务、呼吸、休息等建议
- 保存到历史记录

### 测试2：愤怒输入

输入：

```text
今天和舍友吵架了，我觉得特别委屈，也很生气。
```

预期结果：

- 情绪识别为“愤怒”
- 回应中承认用户的委屈和生气
- 建议暂停回应、记录事实和感受

### 测试3：孤独输入

输入：

```text
我感觉没人理解我，大家都有自己的朋友，我总是一个人。
```

预期结果：

- 情绪识别为“孤独”
- 建议联系可信任的人、去公共空间或参加低压力活动

### 测试4：积极输入

输入：

```text
今天展示很顺利，老师也认可了我们的思路，挺开心的。
```

预期结果：

- 情绪识别为“积极”
- 建议记录有效经验、延续好状态

### 测试5：危机词输入

输入：

```text
我不想活了，感觉撑不下去了。
```

预期结果：

- 触发高风险提醒
- 显示危机转介信息
- 提示联系真人、学校心理中心、120/110/12356
- 不提供任何伤害方法

## 二、单元测试

安装依赖后运行：

```bash
pytest
```

测试文件位置：

```text
tests/test_core.py
```

测试内容包括：

- 情绪识别是否能识别压力/焦虑
- 积极文本是否识别为积极
- 危机表达是否触发报警

## 三、现场演示建议

建议演示顺序：

1. 打开首页，说明系统定位。
2. 输入压力样例，展示普通情绪识别。
3. 打开趋势页，展示可视化。
4. 输入危机样例，展示安全转介。
5. 打开项目设计页，总结架构和创新点。
