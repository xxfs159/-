# 答辩讲稿

各位老师、同学大家好，我们小组的项目是“AI 陪聊情绪树洞：情绪识别与心理资源转介助手”。

我们选择这个项目，是因为大学生在期末考试、课程项目、保研考研、实习就业、人际关系等方面都会遇到压力。但很多时候，大家不一定愿意马上找老师或朋友倾诉，所以我们希望做一个低门槛的匿名树洞，让用户可以先把情绪表达出来。

我们的系统主要完成六个功能。第一，用户可以匿名输入自己的烦恼或心情。第二，系统会识别情绪类型，比如焦虑、低落、愤怒、孤独、压力、平静和积极。第三，系统会生成温和、共情、非评判的回应。第四，系统会保存情绪记录。第五，它会给出一些自我照护建议，比如呼吸练习、短暂休息、写日记或联系朋友。第六，如果系统检测到自伤、自杀或极端绝望等高风险表达，会触发危机词报警，并建议用户联系现实中的人、学校心理中心或紧急服务。

技术实现上，我们使用 Streamlit 搭建 Web 界面，使用 Python 完成情绪识别和危机检测，使用 SQLite 保存历史记录，使用 Pandas 和 Plotly 做趋势可视化。情绪识别部分采用词典和规则加权的方法。这样做的优点是本地可运行、结果可解释、现场展示稳定。系统也预留了 DeepSeek API 接口，如果配置 API Key，就可以由大模型生成更自然的陪伴式回应；如果没有 API Key，本地模板也可以正常运行。

我们的一个重点设计是安全边界。这个系统不是心理医生，也不能替代心理咨询。因此当出现高风险表达时，系统不会继续普通聊天，而是优先提醒用户保护现实安全，并建议联系身边可信任的人、辅导员、学校心理中心或拨打 120、110、12356 等资源。

接下来我演示一下系统。首先输入一句：“最近期末考试和项目 DDL 都挤在一起，我晚上总是睡不着。”系统会识别为焦虑或压力，并显示置信度和判断依据。然后它会生成一段陪伴式回应，并给出拆分任务、短暂休息、呼吸练习等建议。接着我们打开趋势页面，可以看到情绪类型分布图和历史记录。最后输入一个高风险样例，可以看到系统触发危机提醒和资源转介。

本项目的创新点主要有三个。第一，我们把匿名树洞、情绪识别、陪伴回应、趋势记录和危机转介整合成一个闭环。第二，我们强调可解释 AI，系统不只给标签，还会说明判断依据。第三，我们特别考虑了心理应用中的安全问题，明确 AI 的角色是陪伴和转介，而不是诊断和治疗。

当然，项目也有不足。当前情绪识别主要依赖关键词，对复杂语境理解有限；危机词检测可能存在误报和漏报；本地模板的自然度也不如真正的大模型。未来我们计划使用更多大学生情绪语料训练分类模型，接入学校心理资源 RAG 知识库，加入多轮对话记忆，并进一步加强隐私保护。

以上就是我们的项目介绍，谢谢大家。
