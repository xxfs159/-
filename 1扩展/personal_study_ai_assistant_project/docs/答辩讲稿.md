# 现场答辩讲稿

各位老师、同学大家好，我们小组展示的项目是“个人学习档案 AI 助理”。

## 1. 项目背景

我们发现大学生在学习过程中会产生很多数据，比如课程成绩、刷题记录、错题、课堂笔记和学习时长。但是这些数据通常分散在不同平台，学生很难系统地回答“我到底哪里薄弱”“下周该学什么”这样的问题。因此我们希望做一个自动化学习复盘助手，把这些数据统一管理起来，并利用 AI 生成个性化复盘报告。

## 2. 项目目标

本项目的目标是构建一个可持续更新的个人学习档案系统。系统可以聚合学习数据，分析刷题正确率和学习时长，识别薄弱知识点，最后自动生成周度学习复盘报告，并支持自然语言查询。

## 3. 技术方案

我们的系统采用 Streamlit、SQLite、Pandas 和 DeepSeek API 实现。

首先，用户可以在网页中录入刷题记录、学习笔记和学习时长。  
然后，这些数据会存入 SQLite 数据库，形成个人学习档案。  
接着，分析模块会用 Pandas 计算课程正确率、学习时长趋势和知识点薄弱分。  
最后，报告生成模块会调用 DeepSeek 生成周报。如果没有 API Key，系统也可以使用本地规则模板生成报告，保证现场演示稳定。

## 4. 核心算法

系统最核心的部分是薄弱知识点识别。我们没有简单地按正确率排序，而是设计了一个可解释的薄弱分：

薄弱分 = 0.45 × 正确率不足 + 0.25 × 错题数量 + 0.20 × 题目难度 + 0.10 × 笔记不足。

这样做的原因是，一个知识点是否薄弱，不仅取决于做错比例，还和错题数量、题目难度、是否有复盘笔记有关。

## 5. 系统演示流程

下面我们展示系统界面。

第一步，在学习画像看板中，可以看到本周学习时长、刷题数量、正确率和平均专注度。  
第二步，系统会展示每门课的综合画像，比如平均成绩、刷题数量和学习时长。  
第三步，系统会自动识别薄弱知识点 Top 5。  
第四步，我们可以新增一条错题记录，说明系统支持持续更新学习档案。  
第五步，点击生成 AI 周报，系统会输出 Markdown 格式的周度复盘报告。  
第六步，在自然语言查询中输入“我下周应该重点学什么”，系统会根据薄弱点生成学习建议。

## 6. 项目创新点

第一个创新点是学习档案统一建模。我们把成绩、刷题、笔记和学习时长整合到同一个数据库中。  
第二个创新点是可解释诊断。系统不仅告诉用户哪里薄弱，还能说明为什么薄弱。  
第三个创新点是 AI 与规则结合。配置 API Key 时使用大模型生成自然语言周报；没有 API Key 时也能本地运行。  
第四个创新点是闭环学习。系统不是单纯展示图表，而是形成记录、分析、诊断、计划、复盘的完整流程。

## 7. 不足与展望

目前系统的数据主要依赖手动录入，后续可以接入教务系统、刷题平台或学习通，实现自动同步。笔记部分也可以进一步做向量化，扩展为个人 RAG 知识库。学习计划方面，后续可以加入日历约束和任务优先级，使计划更精细。

## 8. 总结

总的来说，本项目实现了一个可运行、可展示、可扩展的个人学习档案 AI 助理。它能够帮助学生更清楚地了解自己的学习状态，发现薄弱点，并制定下一步学习计划。我的汇报结束，谢谢大家。
