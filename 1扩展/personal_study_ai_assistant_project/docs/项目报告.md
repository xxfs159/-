# 个人学习档案 AI 助理项目报告

## 一、项目背景

大学生的学习数据通常分散在不同地方：课程成绩在教务系统中，刷题记录在各类平台中，学习笔记保存在 Markdown、Word 或纸质笔记中，学习时长又可能记录在番茄钟软件里。这种分散状态导致学生很难系统地回答以下问题：

- 我本周到底学了多久？
- 哪门课、哪个知识点最薄弱？
- 错题主要是因为概念不熟、审题不清，还是边界条件遗漏？
- 下周应该优先学什么？
- 能否自动生成一份复盘报告？

因此，本项目设计并实现一个“个人学习档案 AI 助理”，通过结构化数据存储、统计分析和大模型文本生成，帮助学生形成持续复盘的学习闭环。

## 二、项目目标

项目目标是构建一个自动化、个性化、可持续更新的学习复盘助手。系统应具备以下能力：

1. 聚合课程成绩、刷题记录、学习笔记和学习时长。
2. 使用 SQLite 建立个人学习档案数据库。
3. 统计课程成绩趋势、刷题正确率和学习时长变化。
4. 自动识别薄弱知识点和学习习惯问题。
5. 调用 DeepSeek 或本地规则模板生成周度学习复盘报告。
6. 使用 Streamlit 提供可视化展示和聊天式自然语言查询。

## 三、需求分析

### 3.1 功能需求

| 编号 | 功能 | 说明 |
|---|---|---|
| F1 | 学习数据录入 | 支持录入刷题记录、学习笔记和学习时长 |
| F2 | 数据库存储 | 使用 SQLite 保存课程、成绩、错题、笔记和学习时长 |
| F3 | 可视化分析 | 展示课程正确率、学习时长趋势、薄弱点排行榜 |
| F4 | 薄弱点识别 | 根据正确率、错题数、难度和笔记数量计算薄弱分 |
| F5 | AI 周报生成 | 自动生成 Markdown 格式的学习复盘报告 |
| F6 | 自然语言查询 | 支持询问“我哪里薄弱”“下周怎么学”等问题 |

### 3.2 非功能需求

- 系统应易于部署，适合作业展示。
- 即使没有 DeepSeek API Key，也应能完整运行。
- 分析逻辑要可解释，便于答辩说明。
- 数据库结构要清晰，便于后续扩展。

## 四、系统总体设计

系统采用轻量级 Web 应用架构：

```text
用户
 ↓
Streamlit 交互界面
 ↓
数据录入 / 可视化 / 周报生成 / 聊天查询
 ↓
分析模块 Pandas
 ↓
SQLite 个人学习档案数据库
 ↓
DeepSeek API 或本地规则模板
```

### 4.1 模块划分

1. **数据采集模块**  
   通过 Streamlit 表单录入刷题记录、学习时长和学习笔记。

2. **数据存储模块**  
   使用 SQLite 存储结构化数据，包含课程表、成绩表、刷题记录表、笔记表、学习时长表和周报表。

3. **数据分析模块**  
   使用 Pandas 对数据进行聚合统计，生成课程画像、知识点画像和学习习惯摘要。

4. **AI 报告生成模块**  
   如果配置 DeepSeek API Key，则调用大模型生成周报；否则使用本地规则模板生成报告。

5. **交互展示模块**  
   使用 Streamlit 实现学习看板、数据录入、AI 周报和自然语言查询。

## 六、核心算法设计

### 6.1 薄弱知识点识别算法

系统按知识点统计以下指标：

- 正确率 accuracy
- 错题数量 wrong_count
- 平均难度 avg_difficulty
- 笔记数量 note_count

薄弱分计算公式为：

```text
weakness_score =
0.45 × (1 - 正确率)
+ 0.25 × 错题数量归一化
+ 0.20 × 难度归一化
+ 0.10 × 笔记不足程度
```

这个公式体现了四个判断逻辑：

1. 正确率越低，知识点越薄弱。
2. 错题数量越多，说明问题越稳定存在。
3. 难题错误更能反映理解深度不足。
4. 笔记越少，说明复盘材料越不足。

### 6.2 学习习惯分析

系统统计：

- 有学习记录的天数
- 平均单次学习时长
- 平均专注度
- 学习时长趋势

根据规则给出习惯建议：

- 单次学习少于 25 分钟：建议使用番茄钟。
- 平均专注度低于 3.5：建议减少手机干扰。
- 每周学习记录少于 5 天：建议提高学习频率。
- 否则说明学习习惯较稳定。

### 6.3 下周计划生成

系统选取薄弱分最高的知识点，为每个知识点生成任务：

```text
重读笔记 20 分钟 + 完成 5 道同类题 + 整理 1 条错因
```

最后安排一次周六错题分类和周日综合测试，形成闭环。

## 七、AI 报告生成设计

本项目的 AI 报告生成采用“双模式”：

1. **DeepSeek 模式**  
   将课程概况、薄弱知识点、本周指标、学习习惯和下周计划整理为 Prompt，调用 DeepSeek 生成自然语言周报。

2. **本地规则模式**  
   如果未配置 API Key，系统自动使用模板生成 Markdown 周报。这样可以保证在答辩现场没有网络或没有 API Key 时仍然可演示。

## 八、系统实现说明

### 8.1 app.py

Streamlit 主程序，负责页面布局、图表展示、表单录入、周报生成和聊天查询。

### 8.2 database.py

负责 SQLite 连接、建表、课程新增、刷题记录新增、学习时长新增和笔记新增。

### 8.3 analysis.py

负责核心数据分析，包括课程画像、知识点画像、薄弱点识别、学习习惯分析和下周计划生成。

### 8.4 report_generator.py

负责生成学习周报，支持 DeepSeek API 和本地规则模板。

### 8.5 chatbot.py

负责自然语言查询。没有 API Key 时使用关键词匹配回答；有 API Key 时调用 DeepSeek。

## 十、项目创新点

1. **学习档案统一建模**  
   将成绩、刷题、笔记和学习时长统一存储，形成完整学习画像。

2. **可解释薄弱点诊断**  
   薄弱点不是单纯按正确率排序，而是综合考虑正确率、错题数、难度和笔记数量。

3. **AI 与规则结合**  
   既能展示大模型能力，也能保证无 API Key 时可运行、可复现。

4. **闭环学习流程**  
   系统不是只展示数据，而是形成“记录—分析—诊断—计划—复盘”的闭环。

## 十一、不足与改进方向

目前系统仍有以下不足：

1. 数据主要依赖手动录入，后续可接入刷题平台和课程系统。
2. 笔记内容还没有向量化，后续可扩展为 RAG 知识库问答。
3. 学习计划目前以规则生成，后续可加入日历约束和任务优先级。
4. 可加入用户账号系统，实现多用户学习档案管理。

## 十二、总结

本项目完成了一个面向大学生的个人学习档案 AI 助理。系统使用 SQLite 建立学习档案，使用 Pandas 进行学习数据分析，使用 Streamlit 实现交互展示，并结合 DeepSeek 或本地规则模板生成周度学习复盘报告。项目具有较强的实用性、可解释性和可扩展性，适合作为人工智能引论课程的大项目进行展示。
