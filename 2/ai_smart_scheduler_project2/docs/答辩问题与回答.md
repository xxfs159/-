# 答辩问题与回答

## Q1：你们的项目哪里体现了人工智能？

答：项目主要体现在三个方面。第一，自然语言处理模块能够从用户输入的中文任务描述中提取 DDL、耗时、优先级和任务类型。第二，规划型 Agent 模块能够根据课程表、空闲时间和任务紧急程度自动调度学习计划。第三，系统提供可选 DeepSeek 大模型接口，用于生成学习建议、风险提醒和复盘文字。

## Q2：为什么没有完全依赖大模型生成计划？

答：课程表和 DDL 调度对准确性要求较高。如果完全依赖大模型，可能出现时间冲突或计划不可复现的问题。因此我们采用“规则规划 + 可选大模型总结”的方案。规则规划负责可靠调度，大模型负责自然语言总结，这样更稳定，也更适合现场演示。

## Q3：任务优先级如何计算？

答：系统综合用户设置的优先级和距离 DDL 的紧急程度。简化公式为 priority * 10 + urgency * 100，其中 urgency 是剩余小时数的倒数。这样可以让又重要又临近截止的任务优先被安排。

## Q4：如何避免计划与课程冲突？

答：系统先根据课程表生成课程占用时间段，再从用户空闲时间中扣除这些占用时间，得到真实可安排的时间槽。后续所有任务只会安排到这些时间槽中，因此不会与课程冲突。

## Q5：如果任务太多，系统怎么处理？

答：如果可用时间不足，系统不会强行生成虚假计划，而是给出 warning，说明哪些任务没有完全安排，以及已安排多少小时、还缺多少小时。用户可以增加空闲时间、调整任务优先级或减少预计耗时后重新生成。

## Q6：项目的创新点是什么？

答：创新点包括：课程表、DDL 和空闲时间统一建模；支持中文自然语言任务输入；使用规划型 Agent 主动生成学习计划；任务拆分后更符合真实学习习惯；规则算法与大模型总结结合，兼顾稳定性和智能化。

## Q7：未来如何接入飞书 MCP？

答：可以将当前生成的计划表转换为飞书日历事件或任务清单，然后通过飞书开放接口或 MCP 工具调用完成创建日程、消息提醒和任务同步。当前项目已经保留了导出 CSV/Markdown 的接口，后续可在此基础上接入。

## Q8：系统的不足是什么？

答：目前中文时间解析主要基于规则，对非常复杂或模糊的表达支持有限；任务耗时依赖用户估计；还没有形成学习完成打卡和历史行为分析闭环。后续可以接入更强的 NLP 模型、日历工具和个性化推荐算法。
