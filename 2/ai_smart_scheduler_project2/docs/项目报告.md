# AI 智能课表与学习计划生成系统项目报告

## 一、项目背景

大学生在日常学习中经常同时面对课程、作业、考试、项目和社团活动等多种事务。传统手工计划容易出现三个问题：第一，任务信息分散，容易遗漏 DDL；第二，计划安排缺少优先级，常常临近截止时间才开始赶工；第三，课程表、空闲时间和个人学习目标之间没有形成统一协调。

因此，本项目设计并实现一个“AI 智能课表与学习计划生成系统”。用户只需导入课程表、输入作业/考试 DDL、设置空闲时间与学习目标，系统即可自动分析任务优先级，并生成一周精细化学习计划。

## 二、需求分析

### 2.1 功能需求

1. 课程表导入  
   支持 CSV 格式课程表，记录课程名称、上课时间、地点和教师。

2. 任务/DDL 导入  
   支持结构化 CSV 输入，也支持自然语言输入任务。

3. 自然语言解析  
   从中文文本中提取任务名称、截止时间、预计耗时、优先级和任务类型。

4. 智能计划生成  
   根据课程占用、空闲时间、任务优先级和截止时间生成一周计划。

5. 可视化展示  
   展示任务表、计划表、每日学习时长统计和无法安排的任务提示。

6. AI 学习建议  
   支持可选 DeepSeek API 生成本周学习重点、风险提醒和执行建议。

7. 飞书 MCP 同步模拟  
   将计划转换为日历事件 JSON，作为后续接入飞书 MCP 的数据接口。

### 2.2 非功能需求

1. 易运行：使用 Python + Streamlit，适合课程作业演示。
2. 易理解：算法以规则规划为主，便于解释和答辩。
3. 可扩展：后续可接入飞书 MCP、日历同步、消息推送和大模型 Agent。

## 三、总体设计

### 3.1 系统架构

系统采用四层结构：

1. 数据输入层  
   包括课程表、任务表、空闲时间表和自然语言任务输入。

2. NLP 解析层  
   使用正则表达式和规则模板识别日期、星期、时间、耗时、优先级和任务类型。

3. 规划 Agent 层  
   将学习任务建模为待调度任务，将课程和空闲时间建模为时间槽，结合 DDL 和优先级进行调度。

4. 展示与导出层  
   通过 Streamlit 展示表格、柱状图、AI 总结，并支持导出 CSV、Markdown 和飞书 MCP 模拟 JSON。

### 3.2 数据流

用户输入数据后，系统首先读取 CSV 或文本；自然语言任务经过 `nlp_parser.py` 转为结构化任务；随后 `planner.py` 生成可执行计划；最后在 `app.py` 页面中展示和导出。

## 四、核心算法设计

### 4.1 中文任务解析

系统支持以下表达：

- “下周三 23:59”
- “5月20日 20:00”
- “明天晚上10点”
- “预计6小时”
- “很重要”“紧急”“一般”

解析结果包括：

- task_name：任务名称
- deadline：截止时间
- estimated_hours：预计耗时
- priority：优先级
- task_type：任务类型

### 4.2 时间槽建模

系统把一周拆分为多个可用时间槽。课程表表示固定占用时间，空闲时间表示可安排学习的时间。系统会从空闲时间中扣除课程占用，得到真实可用时间。

### 4.3 任务优先级评分

任务排序综合考虑：

1. 用户设置的优先级；
2. 距离截止时间的紧急程度；
3. 任务预计耗时。

简化评分公式为：

```text
score = priority * 10 + urgency * 100
urgency = 1 / 剩余小时数
```

分数越高的任务越早被安排。

### 4.4 任务拆分

为了避免一次学习时间过长，系统将大任务拆分成 30-120 分钟的小学习块。例如 6 小时项目任务会被拆成多个 2 小时左右的计划项。

### 4.5 冲突规避

生成计划时，系统只会把任务安排到：

1. 当前时间之后；
2. 任务 DDL 之前；
3. 用户空闲时间内；
4. 与课程时间不冲突的位置。

如果可用时间不足，系统会给出警告，提示任务无法完全安排。

## 五、系统实现

### 5.1 主界面

`app.py` 使用 Streamlit 实现，包括四个页面：

1. 数据预览；
2. 自然语言输入；
3. 智能计划；
4. 导出与总结。

### 5.2 NLP 模块

`src/nlp_parser.py` 实现时间、耗时、优先级和任务类型识别。该模块虽然不是深度学习模型，但体现了 NLP 中的信息抽取思想，适合作为课程项目的可解释基础版本。

### 5.3 规划模块

`src/planner.py` 是系统核心，负责构造时间槽、扣除课程占用、任务排序、任务拆分和计划生成。

### 5.4 大模型模块

`src/deepseek_client.py` 提供可选大模型调用。如果没有 API Key，系统会自动使用本地模板总结，保证现场展示稳定。

## 六、实验与测试

### 6.1 测试数据

项目提供三份示例数据：

1. `sample_courses.csv`：包含人工智能引论、高等数学、数据结构等课程；
2. `sample_tasks.csv`：包含项目开发、实验报告、复习和展示任务；
3. `sample_free_slots.csv`：包含一周空闲时间。

### 6.2 测试结果

在示例数据下，系统可以生成一周学习计划，并统计每天学习时长。对于可用时间不足的任务，系统会输出无法完全安排的原因。

### 6.3 结果分析

系统生成的计划具有以下特点：

1. 任务按 DDL 和优先级排序；
2. 项目任务会自动拆分；
3. 课程时间不会被占用；
4. 结果可导出，便于复制到飞书、Notion 或课程报告中。

## 七、创新点

1. 将课程表、DDL 和空闲时间统一建模，形成完整学习规划闭环。
2. 支持中文自然语言任务输入，降低用户使用门槛。
3. 引入规划型 Agent 思路，使系统不仅展示信息，还能主动安排任务。
4. 采用“规则规划 + 可选大模型”的架构，兼顾稳定性和 AI 能力展示。
5. 计划结果具有可解释性，方便课程答辩说明。

## 八、不足与改进

当前系统仍有一些不足：

1. 中文时间解析依赖规则，对复杂表达支持有限；
2. 没有真正接入飞书 MCP 和消息推送；
3. 任务耗时依赖用户估计，缺少历史学习行为反馈；
4. 计划调整还需要手动重新生成。

后续可以加入：

1. 飞书日历同步；
2. 学习完成打卡；
3. 用户习惯建模；
4. 强化学习或约束优化算法；
5. 更完整的大模型 Agent 工作流。

## 九、总结

本项目实现了一个面向大学生的 AI 智能课表与学习计划生成系统。系统具备课程表导入、自然语言任务解析、智能计划生成、学习时长统计和 AI 总结等功能，能够较好地满足课程项目中对 NLP、智能体、大模型和可视化展示的要求。项目实现难度适中、逻辑清晰、可运行可演示，适合作为人工智能引论课程大项目提交。
