## 现场展示建议

1. 提前运行一次系统，确认依赖安装成功。
2. 如果没有 DeepSeek API Key，不影响展示，本地规则报告也能运行。
3. 展示时重点讲清楚薄弱分公式和学习闭环。
4. 海报中建议突出 3 个亮点：
   - 统一学习档案
   - 可解释薄弱点识别
   - AI 周报与学习计划生成
5. 答辩时准备回答以下问题：
   - 为什么使用 SQLite？
   - DeepSeek 不可用怎么办？
   - 薄弱分公式为什么这样设计？
   - 项目后续如何扩展？
