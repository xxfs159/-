# 海报内容草稿：具身 AI 桌面宠物

## 标题

基于 BongoCat 的具身 AI 桌面宠物

## 一句话简介

让桌面宠物不只会“卖萌”和同步键鼠，而是能看见用户状态、听懂用户表达，并用语音、动作和表情主动陪伴。

## 研究动机

现有桌宠多以动画和装饰为主，交互方式有限；AI 助手多以聊天窗口为主，缺少可感知、可陪伴的实体感。本项目融合 BongoCat 的桌面宠物形态与多模态 AI，让助手具有更自然的人机互动体验。

## 系统架构

```text
键鼠/摄像头/麦克风
        |
本地感知层：亮度、动作、人脸、语音
        |
Agent 决策层：状态机 + 多模态大模型
        |
输出层：宠物动作、聊天气泡、TTS 语音
        |
桌面端：BongoCat/Tauri 透明置顶窗口
```

## 关键技术

- BongoCat/Tauri：透明桌面窗口、跨平台打包、键鼠动作同步。
- 视觉感知：摄像头帧采样、人脸检测、动作强度和低光判断。
- 多模态模型：Qwen-VL 用于画面理解，DeepSeek/Qwen 用于对话生成。
- 语音交互：ASR 识别用户语音，TTS 生成宠物语音。
- Agent 状态机：把用户行为和视觉信号转化为宠物动作与回复策略。

## 创新点

1. 从桌面宠物扩展到具身 AI Agent。
2. 用本地轻量感知保护隐私，只在用户授权时调用大模型。
3. 将键鼠、视觉、语音和对话统一到一个可解释状态机。
4. 可演示、可迁移、可扩展，适合后续做完整桌面应用。

## 展示效果

- 用户敲键盘时，宠物同步敲击。
- 用户开启摄像头后，宠物根据亮度、动作和人脸状态改变反应。
- 用户询问“你现在看到什么状态”，宠物引用传感指标回答。
- 用户低光或久坐时，宠物给出温和提醒。
- 接入视觉模型后，宠物可根据截图理解场景并聊天。

## 评测方法

| 维度 | 方法 |
| --- | --- |
| 技术创新性 | 对比传统桌宠和普通聊天助手，突出具身感知能力 |
| 技术有效性 | 测试摄像头、输入、对话、语音闭环 |
| 视觉美观性 | 统一桌宠视觉、状态面板和海报结构 |
| 现场汇报质量 | 先演示原型，再讲架构和未来扩展 |

## 后续方向

- 迁移到 BongoCat 源码，复用 Tauri 桌面窗口和输入监听。
- 接入 Live2D 模型，使动作和表情更自然。
- 接入 Qwen-VL 与 DeepSeek，实现真实多模态互动。
- 增加学习陪伴、久坐提醒、日程提醒、长期记忆和个性化角色设定。
