# 《人工智能引论》大项目报告  
## 项目5：智能简历优化与岗位推荐平台

### 一、项目背景

大学生在求职过程中常遇到三个问题：

1. 不清楚自己的简历适合哪些岗位；
2. 不知道简历与岗位 JD 的差距在哪里；
3. 针对不同岗位修改简历、写求职信和准备面试成本较高。

因此，本项目设计并实现一个“智能简历优化与岗位推荐平台”。系统通过自然语言处理、岗位知识库检索、相似度匹配和生成式文本优化，为用户提供岗位推荐、简历修改建议、定制化求职信和面试要点建议。

### 二、项目目标

项目目标是实现一个可以演示、可以复现、适合作业提交的 AI 求职辅助系统。核心功能包括：

- 支持上传或粘贴个人简历；
- 自动抽取简历中的技能、项目、教育背景等信息；
- 构建岗位 JD 知识库；
- 对简历和岗位 JD 进行向量化匹配；
- 输出岗位匹配度、匹配技能和缺失技能；
- 自动生成简历摘要优化、项目经历优化、求职信和面试准备要点；
- 通过 Streamlit 提供可交互演示界面。

### 三、需求分析

#### 3.1 用户需求

用户主要是正在求职或准备实习的学生。用户希望系统能够：

- 快速判断自己适合哪些岗位；
- 明确简历中哪些内容需要补充；
- 得到可以直接参考的求职信和面试准备建议；
- 不需要复杂配置即可完成演示。

#### 3.2 功能需求

| 模块 | 功能 |
|---|---|
| 简历输入模块 | 上传 txt/pdf/docx 或使用示例简历 |
| 简历解析模块 | 抽取姓名、邮箱、电话、技能、项目关键词 |
| 岗位知识库模块 | 保存岗位名称、公司、地点、薪资、技能要求、JD |
| RAG 检索模块 | 将简历作为 Query，岗位 JD 作为知识库进行相似度检索 |
| 匹配评分模块 | 结合语义相似度、技能重合度、求职方向计算综合分 |
| 生成模块 | 生成简历摘要、优化建议、求职信、面试要点 |
| 展示模块 | 通过 Streamlit 展示推荐结果和可解释分析 |

### 四、系统总体设计

系统整体流程如下：

```text
用户上传简历
    ↓
文本解析与清洗
    ↓
技能与基础信息抽取
    ↓
加载岗位 JD 知识库
    ↓
简历与岗位文本向量化
    ↓
语义相似度 + 技能重合度 + 方向匹配综合评分
    ↓
输出 Top-K 岗位推荐
    ↓
生成简历优化建议、求职信、面试要点
```

### 五、核心技术实现

#### 5.1 简历文本解析

系统支持三种输入：

- TXT：直接读取文本；
- PDF：使用 `pypdf` 提取文字；
- DOCX：使用 `python-docx` 提取段落文字。

解析后，系统会进行基础清洗，包括去除多余空白、统一格式等。

#### 5.2 技能关键词抽取

本项目采用可解释的技能词典匹配方法。技能库包括 Python、SQL、NLP、RAG、Prompt工程、Streamlit、LangChain、数据分析等常见求职技能。

选择词典法的原因：

- 适合课程展示，逻辑清晰；
- 结果可解释，便于答辩；
- 不依赖复杂模型，容易复现；
- 后续可扩展为命名实体识别模型或 LLM 信息抽取。

#### 5.3 岗位 JD 知识库

系统使用 `data/jobs.csv` 保存岗位信息，每条岗位包含：

- 岗位编号；
- 岗位名称；
- 公司名称；
- 城市；
- 薪资；
- 方向；
- 技能要求；
- 岗位描述；
- 任职要求。

该文件相当于一个轻量级岗位知识库。实际应用中可以通过爬虫或招聘平台 API 自动更新。

#### 5.4 RAG 检索增强思想

RAG 的核心思想是：先从外部知识库中检索相关内容，再基于检索结果生成答案。

在本项目中：

- 外部知识库：岗位 JD 数据库；
- Query：用户简历；
- 检索目标：与简历最相关的岗位；
- 生成内容：简历优化建议、求职信、面试准备要点。

为了便于本地运行，系统默认使用 TF-IDF 字符 n-gram 向量化和余弦相似度。虽然这不是最强的 embedding 模型，但完整体现了向量检索的基本流程。

后续可替换为：

- BGE embedding；
- Sentence-BERT；
- FAISS / Chroma 向量数据库；
- DeepSeek 生成式优化。

#### 5.5 综合匹配评分

系统不是只看文本相似度，而是融合三个因素：

```text
综合匹配分 = 0.55 × 语义相似度
          + 0.35 × 技能重合度
          + 0.10 × 求职方向匹配度
```

其中：

- 语义相似度：简历文本与岗位 JD 的余弦相似度；
- 技能重合度：简历技能与岗位技能的交集比例；
- 方向匹配度：用户选择的求职方向是否与岗位方向一致。

这种设计兼顾语义相关性、硬技能匹配和用户主观偏好。

#### 5.6 生成式优化模块

系统生成四类内容：

1. 匹配差距分析；
2. 简历个人优势摘要；
3. 项目经历优化建议；
4. 定制化求职信；
5. 面试准备要点。

默认情况下，系统使用本地模板生成，保证无 API Key 时也可演示。若配置 `DEEPSEEK_API_KEY`，系统会调用 DeepSeek API 生成更自然的内容。

### 六、系统功能展示

#### 6.1 简历解析结果

系统可识别：

- 基础信息：姓名、邮箱、电话；
- 技能关键词；
- 项目关键词数量；
- 简历条目数量。

#### 6.2 岗位推荐结果

系统输出 Top-K 岗位推荐列表，展示：

- 岗位名称；
- 公司；
- 城市；
- 薪资；
- 方向；
- 匹配度；
- 已匹配技能；
- 缺失技能。

#### 6.3 可解释推荐

系统会说明为什么推荐某岗位，例如：

- 与简历已有技能匹配；
- 岗位方向符合用户意向；
- 项目经历与 JD 描述存在相似性；
- 缺失技能可以通过学习和项目补充。

#### 6.4 简历优化与求职信生成

系统根据目标岗位生成简历摘要、项目经历优化建议和求职信，帮助用户针对不同岗位进行定制化投递。

### 七、测试说明

#### 7.1 功能测试

| 测试项 | 测试方法 | 预期结果 |
|---|---|---|
| 示例简历加载 | 勾选“使用示例简历” | 成功显示解析结果 |
| 简历解析 | 上传 txt 简历 | 能识别技能和基础信息 |
| 岗位匹配 | 点击开始匹配 | 输出 Top-K 岗位 |

#### 7.2 复现性测试

系统使用固定的示例岗位数据和示例简历，因此在相同环境下可以稳定复现推荐结果。

### 八、创新点

1. **从“岗位推荐”扩展到“求职全流程辅助”**  
   不仅推荐岗位，还自动输出简历优化、求职信和面试要点。

2. **可解释匹配机制**  
   推荐结果包含匹配技能、缺失技能和综合评分，而不是只给一个黑盒结果。

3. **轻量化 RAG 实现**  
   不依赖大型模型也能演示 RAG 的核心思想，适合课堂展示和本地复现。

4. **可扩展 DeepSeek 生成能力**  
   默认离线可运行，配置 API 后可进一步提升文本生成质量。

### 九、项目不足与改进方向

#### 9.1 当前不足

- 示例岗位数据量较小；
- TF-IDF 语义理解能力有限；
- 技能抽取采用词典法，无法完全理解上下文；
- 没有接入真实招聘网站实时数据；
- 缺少用户反馈闭环。

#### 9.2 后续改进

- 使用 BGE embedding 替换 TF-IDF；
- 使用 FAISS 或 Chroma 管理岗位向量；
- 接入真实招聘网站爬虫，定时更新岗位知识库；
- 使用 LLM 进行结构化简历解析；
- 增加用户投递反馈，优化推荐排序；
- 增加账号系统，保存用户简历历史版本。

### 十、总结

本项目围绕“智能简历优化与岗位推荐”场景，完成了从简历解析、岗位知识库构建、RAG 检索、匹配度评估到文本生成的完整 AI 应用流程。项目技术路线清晰、运行方式简单、结果可解释，能够较好满足《人工智能引论》课程大项目的展示与答辩要求。
